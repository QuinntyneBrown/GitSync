# L1 - High-Level Requirements

## L1-001: CLI Tool Structure
The tool shall be implemented as a .NET global tool using `System.CommandLine`, structured with a command-per-file pattern where each command is defined in its own dedicated class and file. The tool shall be publishable to NuGet as a global tool.

## L1-002: Sync Loop Execution
The tool shall provide a default sync command that executes a polling loop at a configurable interval (defaulting to 1 second). The loop shall run until a configurable end time is reached. If no end time or duration is specified, the loop shall run for 8 hours from the moment it starts. The loop shall support graceful cancellation via Ctrl+C.

## L1-003: Git Pull Integration
Each loop iteration shall invoke the locally installed `git` executable to pull remote changes into the working repository before checking for local changes.

## L1-004: Change Detection and Auto-Commit
After pulling, the tool shall inspect the repository for uncommitted local changes. If changes are found, the tool shall stage all changes, create a commit with a configurable message, and push the commit to the remote origin.

## L1-005: Configuration
The tool shall expose command-line options for: the target working directory, the loop poll interval, the loop end time or duration, and the commit message template. All options shall have sensible defaults so the tool can be run without any arguments.

## L1-006: Observability
The tool shall use `Microsoft.Extensions.Logging` for structured logging of all operations. The log level shall be configurable. Every iteration, git command invocation, change detection result, and error shall be logged at an appropriate level.

## L1-007: Dependency Injection
The tool shall use `Microsoft.Extensions.DependencyInjection` (via `IHost` / `HostBuilder`) to compose and resolve all services. All services shall be registered via `IServiceCollection` and resolved through the DI container rather than instantiated directly.

## L1-008: NuGet Packaging
The tool shall be correctly configured as a .NET global tool, including all required NuGet package metadata (ID, version, description, authors, tags, license, and repository URL). The package shall be consumable via `dotnet tool install -g <package-id>`.

## L1-009: Resilience and Safety
Git subprocess invocations shall enforce a configurable timeout so a hanging git process cannot block the loop indefinitely. Errors encountered during a single iteration shall be caught, logged, and must not crash the loop. The working directory must be validated to be an existing directory that contains a git repository before the loop starts.
